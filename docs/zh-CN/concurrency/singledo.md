    # singledo

    让同一份工作在并发场景下只执行一次，避免重复计算或重复请求。

    ## 适用场景

    - 缓存回源。
- 热点 key 的重复请求合并。
- 同一初始化过程不希望被多次触发。

    ## 你会接触到什么

    - 适合和缓存、加载器、远程请求逻辑组合。

    ## 使用建议

    - 如果工作结果需要长期复用，常常要和缓存一起设计，而不只是“只执行一次”。
