# i18n

`i18n` 包负责**多语言翻译**：注册译文、按当前 goroutine 语言查询、占位符插值、复数形式、以及大词条量场景下的**低内存存储**。所有查询走 `language` 包的 fallback 链，未命中返回 key 原样，方便发现遗漏译文。

## 适合什么场景

- 接口/CLI/日志需要按用户语言切换文案，且不想到处传 `context.Context` 或 `lang` 参数。
- 译文带占位符（`{name}` / `{0}`），希望统一插值不重复手拼。
- 一个 key 在不同数量下表达形式不同（如 "1 apple" / "2 apples"），需要 CLDR 风格复数。
- 译文条目极多（数十万），希望减少 map 头开销和 GC 扫描压力，用紧凑的 chunk 存储。

## 常用入口

### 注册与查询

- `T(key, args...)` / `TLocale(tag, key, args...)`：按当前/指定语言查译文并插值。
- `Register(tag, key, text)` / `RegisterMap(tag, kv)`：单条 / 批量注册。
- `Load(map[string]map[string]string)`：从 `locale → key → text` 数据批量装载。
- `New(opts...)` 创建独立 `Bundle`，包级函数实际操作全局默认 Bundle。
- 内置 en/zh 示例 key 走 `messages_en.go` / `messages_zh.go` 的 `init()` 注册（**无** build tag），其他语言用 `messages_xx.go + //go:build lang_xx || lang_all`。

### 占位符

- 命名：`"Hello, {name}!"` + `T(key, "name", "world")`。
- 位置：`"{0} + {1}"` + `T(key, 1, 2)`。
- 自动判别：args 形如 `string, value, string, value...` 视为命名，否则位置。
- 不含 `{` 的文本走零分配快路径。

### 复数

- `TPlural(key, n, args...)` / `(*Bundle).TPlural`：按 `n` 选 CLDR category，查 `key.<category>`，缺则回退 `key.other` → `key` 原文。
- 内置规则：en（n==1 → one；其他 → other）、zh（恒 other）。
- `n` 自动注入为 `{n}` 占位符，无需手传。

### 低内存模式

- `New(WithLowMemory(), WithMaxBytes(n))`：切换到 `chunkStore`。
- 64 桶 × 64KB heap chunk，`map[uint64]uint64` 无指针，GC 扫描成本随译文数量近乎不变。
- `hash/maphash` 哈希 + 全 key 比对防冲突；**不驱逐**（译文不可丢），超 `MaxBytes` 提示后继续扩 chunk。

## 使用建议

- 启动时在 `init()` 或入口处用 `Register` / `Load` 装齐所有静态译文；运行时只查询，不混在请求路径里注册。
- 语言切换走 `language.Set`，按请求级 goroutine-local 隔离，不引 `context`。
- 译文条目少（几千以内）默认 `New()` 用 mapStore 即可；过十万级再切 `WithLowMemory()`，并基准对比内存。
- 复数 key 一律拆 `.one` / `.other` 形式，让规则函数自动选择，不要在调用处 `if n == 1`。
- 未命中返回 key 原样是**特性**，便于扫日志查漏；不要靠 panic 暴露缺失。
