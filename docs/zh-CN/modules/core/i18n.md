# i18n

`i18n` 包负责**多语言翻译**：按 locale 加载译文文件（磁盘 / `embed.FS` / `fs.FS`），用 `text/template` 做变量插值，按当前 goroutine 语言（来自 `language` 包）走 fallback 链命中。内置 **JSON / YAML / TOML** 三种格式，支持嵌套 key 自动扁平化（`{"errors":{"not_found":"..."}}` → `errors.not_found`）。

## 适合什么场景

- 翻译文案按 locale 分文件管理（`en.json` / `zh-CN.yaml` / `ja.toml`），独立装载。
- 译文随 binary 发布，用 `embed.FS` 嵌入资源扫描。
- 递归扫描多层目录（每个子模块自带 locale 文件）。
- 启动一次性灌入，运行时只查询；语言切换走 goroutine-local，不传 `context`。

## 常用入口

### 加载

| 入口 | 用途 |
|---|---|
| `LoadFile(path)` | 磁盘单文件，lang 从 basename 推断 |
| `LoadFileWithLang(tag, path)` | 磁盘单文件，显式 lang |
| `LoadFs(fsys, path)` | `fs.FS` 单文件（含 `embed.FS`），lang 从 basename 推断 |
| `LoadFsWithLang(tag, fsys, path)` | `fs.FS` 单文件 + 显式 lang |
| `LoadDir(root)` | 磁盘递归扫描（`fs.WalkDir`） |
| `LoadFsDir(fsys, root)` | `fs.FS` 递归扫描 |
| `LoadLocalizes(fsys)` | `fs.FS` 单层 `localize/` 目录 |
| `LoadLocalizesWithFs(dir, fsys)` | `fs.FS` 单层指定目录 |

文件名约定：`<lang>.<ext>`，例 `en.json` / `zh-CN.yaml`。

格式：内置 `.json` / `.yaml` / `.yml` / `.toml`。扩展用 `RegisterLocalizer(ext, l)`。

### 查询

- `Localize(key, args...)` — 按当前 `language.Get()` 查询并模板插值。
- `LocalizeWithLang(tag, key, args...)` — 指定 locale。
- 未命中 → 返回 key 原样，便于发现遗漏译文。

插值用 `text/template`：`args[0]` 作为模板数据（如 struct / map），译文里写 `{{.Name}}` 引用字段。`AddTemplateFunc(name, fn)` 注册自定义函数。

### 容器与默认实例

- `New(opts...)` 创建独立 `I18n`：`WithDefaultLang(tag)` / `WithTemplateFuncs(funcs)`。
- `Default` 是包级实例，包级 `Localize` / `Register` / `Load*` 全部走它。
- `Register(tag, key, value)` / `RegisterBatch(tag, map)` 也可代码内直接注入译文。

### 语言切换

- `SetLanguage(tag)` / `GetLanguage()` / `DelLanguage()` — 当前 goroutine 语言（薄包装 `language` 包）。
- fallback：当前 lang → 当前 lang 主语言（zh-CN → zh）→ 默认 lang → 默认 lang 主语言。

## 使用建议

- 启动时构造 `I18n`，调 `LoadFsDir(embedFs, "i18n")` 或 `LoadDir("./i18n")` 一次性灌入，再 `SetDefault` 或直接用 `Default`。
- 译文文件用 YAML 写嵌套结构最直观（`errors.not_found.title` 拆三层），JSON / TOML 也支持。
- 模板数据传 struct 而非 map：编译期可检查字段名，性能也好。
- 未命中返回 key 原样是**特性**，扫日志查漏；不要靠 panic 暴露缺失。
- 语言切换在请求入口设一次（如 HTTP middleware 从 `Accept-Language` 解析后 `SetLanguage`），业务代码无需感知。
