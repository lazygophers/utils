# xtime007

`xtime007` 面向的是**全天候制度语义**。相比 955 / 996，这类规则通常更简单，但仍适合通过独立模块表达，而不是在业务里散落特殊判断。

## 适合什么场景

- 需要表达 007 规则下的时间判断。
- 想在同一套时间主题里比较不同制度模型。
- 需要把制度语义与普通时间工具分开。

## 使用建议

- 该模块表达的是制度抽象，而不是对现实团队安排的价值判断。
- 如果你的场景并非固定 007 模型，直接使用它通常并不合适。
- 需要日历、农历或节气信息时，仍应回到 [xtime](/modules/time/xtime)。
