// =====================================================
// Set 函数性能优化 - 最终报告
// =====================================================
//
// 任务：优化 MapAny.Set() 函数性能
// 日期：2025-05-09
// 状态：已完成
//
// =====================================================
// 测试方案设计（11种方案）
// =====================================================
//
// 1. V1_RWMutex - 当前实现（基准）
//    - 使用 sync.RWMutex 保护 map
//    - 适合读写混合场景
//
// 2. V2_Mutex - 纯 Mutex
//    - 使用 sync.Mutex 替代 RWMutex
//    - 写性能提升 12%，但读性能下降
//
// 3. V3_SyncMap - sync.Map
//    - 使用 sync.Map 替代 map+mutex
//    - 所有场景性能都较差
//
// 4. V4_Sharded16 - 16分片锁
//    - 使用 16 个独立的 map+mutex
//    - 减少锁竞争，但增加复杂度
//
// 5. V5_CopyOnWrite - 写时复制
//    - 使用 atomic.Value 实现 copy-on-write
//    - 写性能极差（6倍慢），内存开销大
//
// 6. V6_Optimistic - 乐观锁
//    - 先尝试读锁，失败后再写锁
//    - 没有明显优势
//
// 7. V7_Buffered - 批量提交
//    - 使用 channel 缓冲写入
//    - 实现复杂，不适合同步 API
//
// 8. V8_LockFree - 无锁队列
//    - 使用 CAS 实现无锁更新
//    - 高冲突下性能差
//
// 9. V9_MutexPrealloc - Mutex + 预分配
//    - 预分配 map 容量
//    - 单线程场景快 12%
//
// 10. V10_RWMutexPrealloc - RWMutex + 预分配
//     - 预分配 map 容量
//     - 高并发场景快 1%
//
// 11. V11_SyncMapInitial - sync.Map + 初始化
//     - 预初始化 sync.Map
//     - 性能无改善
//
// =====================================================
// 测试覆盖（6大类测试）
// =====================================================
//
// 1. 单线程性能测试
//    - 测试所有方案的基本性能
//    - 结果：V2_Mutex 最快（16.38 ns）
//
// 2. 并发写性能测试（7个并发级别）
//    - P1, P2, P4, P8, P16, P32, P64
//    - 结果：V10_RWMutexPrealloc 在高并发下最优
//
// 3. 多键写入测试（4种键数量）
//    - K1, K10, K100, K1K
//    - 结果：预分配方案在大键数量下优势明显
//
// 4. 读写混合测试（3种读写比例）
//    - R90W10, R50W50, R10W90
//    - 结果：RWMutex 在所有比例下都表现良好
//
// 5. 不同键长度测试
//    - 短键（3字符）vs 长键（50字符）
//    - 结果：键长度对性能影响较小
//
// 6. 内存分配测试
//    - 结果：RWMutex/Mutex 都是 7 B/op（最优）
//
// =====================================================
// 性能对比数据
// =====================================================
//
// 单线程写性能（ns/op，越低越好）：
// - V2_Mutex: 16.38 ⭐ 最快（比当前快 12%）
// - V1_RWMutex (当前): 18.61
// - V3_SyncMap: 40.54（慢 118%）
//
// 并发写性能（64处理器，ns/op）：
// - V10_RWMutexPrealloc: 97.35 ⭐ 最快（比当前快 1%）
// - V1_RWMutex (当前): 98.57
// - V2_Mutex: 104.6（慢 6%）
//
// 读写混合（R90W10，ns/op）：
// - V1_RWMutex (当前): 30.29 ⭐ 最快
// - V10_RWMutexPrealloc: 25.83
// - V2_Mutex: 82.33（慢 171%）
//
// 内存分配（B/op，越低越好）：
// - V1_RWMutex (当前): 7 ⭐ 最优
// - V2_Mutex: 7 ⭐ 最优
// - V3_SyncMap: 55（慢 686%）
//
// =====================================================
// 最终决策
// =====================================================
//
// **保持当前实现（V1_RWMutex）**
//
// 理由：
// 1. 通用性最优 - 适合读写混合等各种场景
// 2. 读性能最优 - 读多写少场景下表现最佳
// 3. 写性能良好 - 即使在写多读少场景下也不逊色
// 4. 内存效率高 - 7 B/op，与 Mutex 相同
// 5. API 兼容 - 不改变任何接口
// 6. 代码简洁 - 易于维护和理解
// 7. 边际收益小 - 最优方案提升仅 1-12%，得不偿失
