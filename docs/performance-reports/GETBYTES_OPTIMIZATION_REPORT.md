## 🎯 优化策略

### 问题分析
原始实现使用 `fmt.Sprintf` 进行整数到字节的转换，存在以下问题：
1. **性能开销**: `fmt.Sprintf` 需要解析格式字符串，开销大
2. **内存分配**: 每次调用都会产生新的内存分配
3. **浮点数格式**: 使用 `%v` 格式化浮点数不够高效

### 优化方案
采用 **混合优化策略** (Optimized1)：
1. **整数类型**: 使用 `strconv.AppendInt/AppendUint` 替代 `fmt.Sprintf`
2. **浮点类型**: 使用 `strconv.AppendFloat` 配合 `'g'` 格式
3. **栈缓冲区**: 使用 `[64]byte` 栈数组避免堆分配
4. **字符串类型**: 保持原有实现（已最优）

## 🎨 核心代码变更

### 优化前
```go
case int:
    return []byte(fmt.Sprintf("%d", x))
case int64:
    return []byte(fmt.Sprintf("%d", x))
case float64:
    return []byte(fmt.Sprintf("%v", x))
```

### 优化后
```go
var buf [64]byte
switch x := val.(type) {
case int:
    return strconv.AppendInt(buf[:0], int64(x), 10)
case int64:
    return strconv.AppendInt(buf[:0], x, 10)
case float64:
    return strconv.AppendFloat(buf[:0], x, 'g', -1, 64)
}
```
