## 优化前分析

### 原始实现问题
1. **双重边界检查**: 每个分支执行 `index < 0 || index >= len(v)`（2 次比较）
2. **冗余错误格式化**: 每次越界都调用 `fmt.Errorf` 格式化详细错误消息
3. **内存分配**: 错误格式化产生额外的字符串和内存分配

### 代码示例
```go
// 原始实现
case []any:
    if index < 0 || index >= len(v) {  // 双重检查
        return nil, fmt.Errorf("%w: index %d out of range [0, %d)", ErrOutOfRange, index, len(v))
    }
    return v[index], nil
```

## 优化方案

### 策略
**合并边界检查 + 延迟错误格式化**

### 实现代码
```go
// 优化实现
case []any:
    if uint(index) >= uint(len(v)) {  // 合并检查（负数自动转大正数）
        return nil, ErrOutOfRange       // 延迟错误（返回预定义错误）
    }
    return v[index], nil
```

### 优化要点
1. **合并边界检查**:
   - 使用 `uint(index) >= uint(len(v))` 隐式处理负数
   - 负数转 uint 变成极大值，自动触发越界
   - 1 次比较替代 2 次比较

2. **延迟错误格式化**:
   - 直接返回 `ErrOutOfRange` 预定义错误
   - 避免每次越界都格式化消息
   - 减少内存分配

3. **代码简化**:
   - 每个分支减少 1 行代码
   - 更易维护
