# i18n

The `i18n` package handles **multi-language translation**: register messages, query by the current goroutine's language, interpolate placeholders, pluralize, and — for very large catalogs — store everything in a **low-memory** chunk layout. Lookups walk the `language` package fallback chain, and a miss returns the key verbatim so missing translations are easy to spot.

## When to reach for it

- UI / CLI / log output needs to switch by user language without threading `context.Context` or a `lang` argument through every call.
- Messages contain placeholders (`{name}` / `{0}`) and you don't want to hand-format them everywhere.
- A key reads differently by quantity ("1 apple" / "2 apples") and you want CLDR-style plural categories.
- Catalogs are huge (hundreds of thousands of entries) and you want fewer map headers + lower GC scan cost via compact chunks.

## Common entry points

### Register & query

- `T(key, args...)` / `TLocale(tag, key, args...)` — look up by current / specified language and interpolate.
- `Register(tag, key, text)` / `RegisterMap(tag, kv)` — single / bulk registration.
- `Load(map[string]map[string]string)` — bulk load from a `locale → key → text` map.
- `New(opts...)` creates a standalone `Bundle`; package-level functions operate on the global default Bundle.
- Built-in en/zh sample keys live in `messages_en.go` / `messages_zh.go` `init()` (**no** build tag). Other languages use `messages_xx.go + //go:build lang_xx || lang_all`.

### Placeholders

- Named: `"Hello, {name}!"` + `T(key, "name", "world")`.
- Positional: `"{0} + {1}"` + `T(key, 1, 2)`.
- Auto-detect: `string, value, string, value...` → named; otherwise positional.
- Text without `{` takes a zero-allocation fast path.

### Plurals

- `TPlural(key, n, args...)` / `(*Bundle).TPlural` — pick a CLDR category by `n`, look up `key.<category>`, fall back to `key.other` → `key`.
- Built-in rules: en (n==1 → one; else other), zh (always other).
- `n` is injected as the `{n}` placeholder automatically.

### Low-memory mode

- `New(WithLowMemory(), WithMaxBytes(n))` switches to `chunkStore`.
- 64 buckets × 64 KB heap chunks; `map[uint64]uint64` holds no pointers, so GC scan cost stays roughly constant regardless of catalog size.
- `hash/maphash` plus a full-key compare guards against collisions; **no eviction** (translations must not be lost) — exceeding `MaxBytes` keeps appending new chunks.

## Guidelines

- Register all static translations at startup (in `init()` or a setup hook). Don't mix registration into request paths.
- Switch language via `language.Set`; isolation is per-goroutine — no `context` plumbing.
- Small catalogs (a few thousand) are fine on the default mapStore; switch to `WithLowMemory()` only past tens of thousands and benchmark first.
- For plurals, always split keys into `.one` / `.other` forms and let the rule function pick — don't `if n == 1` at the call site.
- A missing key returning verbatim is **by design**: scan logs to find gaps. Don't surface misses via panic.
