# i18n

The `i18n` package handles **multi-language translation**: load message files per locale (from disk, `embed.FS`, or any `fs.FS`), interpolate via `text/template`, and look up by the current goroutine's language (from the `language` package) walking a fallback chain. Built-in formats: **JSON / YAML / TOML**. Nested keys auto-flatten (`{"errors":{"not_found":"..."}}` â†’ `errors.not_found`).

## When to reach for it

- Translation files split per locale (`en.json` / `zh-CN.yaml` / `ja.toml`), loaded independently.
- Messages ship inside the binary via `embed.FS`.
- Recursive scan over multi-level directories (each submodule ships its own locale files).
- Loaded once at startup, query-only at runtime; language switching is goroutine-local â€” no `context` plumbing.

## Common entry points

### Load

| Entry | Purpose |
|---|---|
| `LoadFile(path)` | Single file from disk; lang inferred from basename |
| `LoadFileWithLang(tag, path)` | Single file from disk; explicit lang |
| `LoadFs(fsys, path)` | Single file from any `fs.FS` (incl. `embed.FS`); lang inferred |
| `LoadFsWithLang(tag, fsys, path)` | `fs.FS` single file + explicit lang |
| `LoadDir(root)` | Recursive disk scan via `fs.WalkDir` |
| `LoadFsDir(fsys, root)` | Recursive `fs.FS` scan |
| `LoadLocalizes(fsys)` | One-level `localize/` directory scan on an `fs.FS` |
| `LoadLocalizesWithFs(dir, fsys)` | One-level scan of any directory on an `fs.FS` |

Filename convention: `<lang>.<ext>`, e.g. `en.json` / `zh-CN.yaml`.

Formats: `.json` / `.yaml` / `.yml` / `.toml` built-in. Extend via `RegisterLocalizer(ext, l)`.

### Query

- `Localize(key, args...)` â€” look up by current `language.Get()` and interpolate.
- `LocalizeWithLang(tag, key, args...)` â€” explicit locale.
- A miss returns the key verbatim â€” makes gaps easy to spot.

Interpolation uses `text/template`: `args[0]` is the template data (struct or map); the message references it as `{{.Name}}`. Register custom funcs with `AddTemplateFunc(name, fn)`.

### Container & default instance

- `New(opts...)` creates a standalone `I18n`: `WithDefaultLang(tag)` / `WithTemplateFuncs(funcs)`.
- `Default` is the package-level instance; package-level `Localize` / `Register` / `Load*` all route through it.
- `Register(tag, key, value)` / `RegisterBatch(tag, map)` inject translations from code.

### Language switching

- `SetLanguage(tag)` / `GetLanguage()` / `DelLanguage()` â€” current goroutine's language (thin wrappers over the `language` package).
- Fallback: current lang â†’ current lang's base (zh-CN â†’ zh) â†’ default lang â†’ default lang's base.

## Guidelines

- At startup, build an `I18n`, call `LoadFsDir(embedFs, "i18n")` or `LoadDir("./i18n")` to load everything, then `SetDefault` (or use `Default` directly).
- YAML is the most natural format for nested message trees (`errors.not_found.title` over three levels); JSON / TOML work too.
- Pass a struct (not a map) as template data: compile-time field-name checks, better performance.
- Returning the key verbatim on a miss is **by design** â€” scan logs to find gaps. Don't surface misses via panic.
- Set the language once at the request entry (e.g. HTTP middleware parses `Accept-Language` and calls `SetLanguage`); business code doesn't need to know.
