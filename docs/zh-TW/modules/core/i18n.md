# i18n

`i18n` 套件負責**多語言翻譯**：註冊譯文、按當前 goroutine 語言查詢、佔位符插值、複數形式，以及大詞條量場景下的**低記憶體存儲**。所有查詢走 `language` 套件的 fallback 鏈，未命中回傳 key 原樣，方便發現遺漏譯文。

## 適合什麼場景

- 介面/CLI/日誌需要按使用者語言切換文案，且不想到處傳 `context.Context` 或 `lang` 參數。
- 譯文帶佔位符（`{name}` / `{0}`），希望統一插值不重複手拼。
- 一個 key 在不同數量下表達形式不同（如 "1 apple" / "2 apples"），需要 CLDR 風格複數。
- 譯文條目極多（數十萬），希望減少 map 標頭開銷與 GC 掃描壓力，用緊湊的 chunk 存儲。

## 常用入口

### 註冊與查詢

- `T(key, args...)` / `TLocale(tag, key, args...)`：按當前/指定語言查譯文並插值。
- `Register(tag, key, text)` / `RegisterMap(tag, kv)`：單條 / 批量註冊。
- `Load(map[string]map[string]string)`：從 `locale → key → text` 資料批量裝載。
- `New(opts...)` 建立獨立 `Bundle`，套件級函式實際操作全域預設 Bundle。
- 內建 en/zh 範例 key 走 `messages_en.go` / `messages_zh.go` 的 `init()` 註冊（**無** build tag），其他語言用 `messages_xx.go + //go:build lang_xx || lang_all`。

### 佔位符

- 命名：`"Hello, {name}!"` + `T(key, "name", "world")`。
- 位置：`"{0} + {1}"` + `T(key, 1, 2)`。
- 自動判別：args 形如 `string, value, string, value...` 視為命名，否則位置。
- 不含 `{` 的文本走零分配快路徑。

### 複數

- `TPlural(key, n, args...)` / `(*Bundle).TPlural`：按 `n` 選 CLDR category，查 `key.<category>`，缺則回退 `key.other` → `key` 原文。
- 內建規則：en（n==1 → one；其他 → other）、zh（恆 other）。
- `n` 自動注入為 `{n}` 佔位符，無需手傳。

### 低記憶體模式

- `New(WithLowMemory(), WithMaxBytes(n))`：切換到 `chunkStore`。
- 64 桶 × 64KB heap chunk，`map[uint64]uint64` 無指標，GC 掃描成本隨譯文數量近乎不變。
- `hash/maphash` 雜湊 + 全 key 比對防衝突；**不驅逐**（譯文不可丟），超 `MaxBytes` 提示後繼續擴 chunk。

## 使用建議

- 啟動時在 `init()` 或入口處用 `Register` / `Load` 裝齊所有靜態譯文；執行階段只查詢，不混在請求路徑裡註冊。
- 語言切換走 `language.Set`，按請求級 goroutine-local 隔離，不引 `context`。
- 譯文條目少（幾千以內）預設 `New()` 用 mapStore 即可；過十萬級再切 `WithLowMemory()`，並基準比對記憶體。
- 複數 key 一律拆 `.one` / `.other` 形式，讓規則函式自動選擇，不要在呼叫處 `if n == 1`。
- 未命中回傳 key 原樣是**特性**，便於掃日誌查漏；不要靠 panic 暴露缺失。
