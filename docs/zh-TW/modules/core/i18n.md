# i18n

`i18n` 套件負責**多語言翻譯**：按 locale 載入譯文檔案（磁碟 / `embed.FS` / `fs.FS`），用 `text/template` 做變數插值，按當前 goroutine 語言（來自 `language` 套件）走 fallback 鏈命中。內建 **JSON / YAML / TOML** 三種格式，支援巢狀 key 自動扁平化（`{"errors":{"not_found":"..."}}` → `errors.not_found`）。

## 適合什麼場景

- 翻譯文案按 locale 分檔管理（`en.json` / `zh-CN.yaml` / `ja.toml`），獨立裝載。
- 譯文隨 binary 發布，用 `embed.FS` 嵌入資源掃描。
- 遞迴掃描多層目錄（每個子模組自帶 locale 檔案）。
- 啟動一次性灌入，執行期只查詢；語言切換走 goroutine-local，不傳 `context`。

## 常用入口

### 載入

| 入口 | 用途 |
|---|---|
| `LoadFile(path)` | 磁碟單檔，lang 從 basename 推斷 |
| `LoadFileWithLang(tag, path)` | 磁碟單檔，顯式 lang |
| `LoadFs(fsys, path)` | `fs.FS` 單檔（含 `embed.FS`），lang 從 basename 推斷 |
| `LoadFsWithLang(tag, fsys, path)` | `fs.FS` 單檔 + 顯式 lang |
| `LoadDir(root)` | 磁碟遞迴掃描（`fs.WalkDir`） |
| `LoadFsDir(fsys, root)` | `fs.FS` 遞迴掃描 |
| `LoadLocalizes(fsys)` | `fs.FS` 單層 `localize/` 目錄 |
| `LoadLocalizesWithFs(dir, fsys)` | `fs.FS` 單層指定目錄 |

檔名約定：`<lang>.<ext>`，例 `en.json` / `zh-CN.yaml`。

格式：內建 `.json` / `.yaml` / `.yml` / `.toml`。擴充用 `RegisterLocalizer(ext, l)`。

### 查詢

- `Localize(key, args...)` — 按當前 `language.Get()` 查詢並模板插值。
- `LocalizeWithLang(tag, key, args...)` — 指定 locale。
- 未命中 → 回傳 key 原樣，便於發現遺漏譯文。

插值用 `text/template`：`args[0]` 作為模板資料（如 struct / map），譯文裡寫 `{{.Name}}` 引用欄位。`AddTemplateFunc(name, fn)` 註冊自訂函式。

### 容器與預設實例

- `New(opts...)` 建立獨立 `I18n`：`WithDefaultLang(tag)` / `WithTemplateFuncs(funcs)`。
- `Default` 是套件級實例，套件級 `Localize` / `Register` / `Load*` 全部走它。
- `Register(tag, key, value)` / `RegisterBatch(tag, map)` 也可代碼內直接注入譯文。

### 語言切換

- `SetLanguage(tag)` / `GetLanguage()` / `DelLanguage()` — 當前 goroutine 語言（薄封裝 `language` 套件）。
- fallback：當前 lang → 當前 lang 主語言（zh-CN → zh）→ 預設 lang → 預設 lang 主語言。

## 使用建議

- 啟動時建構 `I18n`，呼叫 `LoadFsDir(embedFs, "i18n")` 或 `LoadDir("./i18n")` 一次性灌入，再 `SetDefault` 或直接用 `Default`。
- 譯文檔案用 YAML 寫巢狀結構最直觀（`errors.not_found.title` 拆三層），JSON / TOML 也支援。
- 模板資料傳 struct 而非 map：編譯期可檢查欄位名，效能也好。
- 未命中回傳 key 原樣是**特性**，掃日誌查漏；不要靠 panic 暴露缺失。
- 語言切換在請求入口設一次（如 HTTP middleware 從 `Accept-Language` 解析後 `SetLanguage`），業務代碼無需感知。
