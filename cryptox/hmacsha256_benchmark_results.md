## 关键发现

### 1. 性能提升范围
- **最佳方案**: 零分配优化 / 手动 hex 编码 (固定数组)
- **性能提升**: 1.14x - 1.30x
- **时间节省**: 约 13-23% (从 ~345-395ns/op 降至 ~302-310ns/op)

### 2. 优化方案分类
- **第一梯队 (302-310ns/op)**: 零分配、固定数组、全局查找表、内联字符串
- **第二梯队 (311-328ns/op)**: 循环展开、unsafe 指针、encoding/hex
- **第三梯队 (345-395ns/op)**: 原始实现 (fmt.Sprintf)

### 3. 性能分析
- **手动 hex 编码** 比 `fmt.Sprintf` 快约 13-30%
- **固定数组** (`[64]byte`) 比切片分配更高效
- **循环展开** 和 **SIMD 模拟** 在 Go 中效果不明显，可能因编译器已优化
- **unsafe 指针** 反而略慢，可能因边界检查优化抵消了收益
- **encoding/hex.EncodeToString** 性能接近手动实现，但代码更简洁

### 4. 内存分配
- **零分配方案**: 使用栈上数组，避免堆分配
- **固定数组**: 编译器可能优化为栈分配
- **切片方案**: 需要堆分配 (make)

## 选择的方案

### 推荐：手动 hex 编码 (固定数组)

**理由**：
1. **性能稳定**: 两次测试都进入前三名
2. **代码简洁**: 易于理解和维护
3. **与项目一致**: 与 HMACSHA1、HMACMd5 优化模式一致
4. **零外部依赖**: 不需要 unsafe 或标准库
5. **编译器友好**: 固定数组便于编译器优化

**代码实现**：
```go
func HMACSHA256[M string | []byte](key, message M) string {
	h := hmac.New(sha256.New, []byte(key))
	_, _ = h.Write([]byte(message))
	hash := h.Sum(nil)

	const hexchars = "0123456789abcdef"
	var result [64]byte // SHA256 = 32 字节
	for i := 0; i < 32; i++ {
		b := hash[i]
		result[i*2] = hexchars[b>>4]
		result[i*2+1] = hexchars[b&0x0f]
	}
	return string(result[:])
}
```
